# playground hard-merge pulls

该目录用于模拟多个 Pull Request，每个 Pull 内包含多个 commit，每个 commit 是同一仓库代码在不同时间点的快照，用于覆盖率合并与变更场景测试。

结构示例：

```
pulls/
  1/
    commits/
      0001/repo/...
      0002/repo/...
      0003/repo/...
  2/
    commits/
      0001/repo/...
      0002/repo/...
      0003/repo/...
  3/
    commits/
      0001/repo/...
      0002/repo/...
      0003/repo/...
```

后续可在每个 `commit` 目录下补充相应的覆盖率产物（如 Istanbul 格式 JSON），用于测试合并逻辑。
