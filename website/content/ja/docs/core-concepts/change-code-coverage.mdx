# 変更コードカバレッジ

変更コードカバレッジにより、今回変更されたコードのテスト状況に焦点を当てることができます。

## 計算ロジック

1. git diffで各ファイルの`追加行`と`削除行`を取得
2. 変更されたファイルのカバレッジデータを走査し、`追加行`が`すべての次元`でカバーされているかどうかを比較
3. 単一ファイルの変更コードカバレッジを計算。式：`(追加行数 - 未カバー行数) / 追加行数`

> [!TIP]
>
> その中で、多くの`追加行`は`ブランチ`、`関数`、`ステートメント`のいずれにも属していませんが、計算に参加し、デフォルトで`カバー済み`とみなされます。
