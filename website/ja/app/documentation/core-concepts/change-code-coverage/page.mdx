# コード変更のカバレッジ

コード変更のカバレッジにより、変更されたコードのテスト状況に焦点を当てることができます。

## 計算ロジック

1. git diffで各ファイルの`追加行`と`削除行`を取得
2. そのファイルのカバレッジデータを走査し、`追加行`が`すべての側面`でカバーされているかを確認
3. 個々のファイルの変更コードカバレッジを計算。計算式：`(追加行数 - 未カバー行数) / 追加行数`

> [!TIP]
>
> なお、`追加行数`の多くは`ブランチ`、`関数`、`ステートメント`のいずれにも属しませんが、計算には含まれ、デフォルトで`カバー済み`とみなされます。
